# Contributing to / Contribuyendo a: Blog Database System 🚀

First of all, thank you for considering contributing to this project! / *En primer lugar, ¡gracias por considerar contribuir a este proyecto!*

---

## 🌍 English Version

### 🛠️ How Can I Contribute?

#### 1. Reporting Bugs
If you find an error in the SQL scripts or database logic:
* Open an **Issue** describing the problem.
* Include the PostgreSQL error message if applicable.
* Specify the PostgreSQL version you are using.

#### 2. Suggesting Enhancements
Do you have an idea to optimize a query or improve the ERD design?
* Create an **Issue** explaining the enhancement.
* If you plan to implement it, please wait for discussion before sending a Pull Request.

#### 3. Pull Requests (PRs)
1. **Fork** the repository.
2. Create a branch for your feature: `git checkout -b feature/NewEnhancement`.
3. If adding new tables, ensure you update the `.sql` script and the ERD diagram.
4. Maintain SQL style: Keywords in uppercase (`SELECT`, `JOIN`) and table/column names in lowercase `snake_case`.
5. **Commit** with a clear message: `git commit -m 'Add categories table'`.
6. **Push** to the branch and open a **Pull Request**.

## 🌎 Versión en Español

### 🛠️ ¿Cómo puedo contribuir?

#### 1. Reportar Errores
Si encuentras un error en los scripts SQL o en la lógica de la base de datos:
* Abre un **Issue** describiendo el problema.
* Incluye el mensaje de error de PostgreSQL si es posible.
* Indica la versión de PostgreSQL que estás utilizando.

#### 2. Sugerir Mejoras
¿Tienes una idea para optimizar una consulta o mejorar el diseño del ERD?
* Crea un **Issue** explicando la mejora.
* Si planeas implementarla, por favor espera a que se discuta antes de enviar un Pull Request.

#### 3. Pull Requests (PRs)
1. Haz un **Fork** del repositorio.
2. Crea una rama para tu mejora: `git checkout -b feature/NuevaMejora`.
3. Si añades nuevas tablas, asegúrate de actualizar el script `.sql` y el diagrama ERD.
4. Mantén el estilo SQL: Palabras clave en mayúsculas (`SELECT`, `JOIN`) y nombres de tablas/columnas en minúsculas y `snake_case`.
5. Haz un **Commit** con un mensaje claro: `git commit -m 'Añadir tabla de categorías'`.
6. Haz **Push** a la rama y abre el **Pull Request**.

## 📝 Style Guide / Guía de Estilo

| Feature / Característica | Standard / Estándar |
| :--- | :--- |
| **SQL Keywords** | Uppercase / Mayúsculas (`SELECT`, `FROM`) |
| **Names / Nombres** | Lowercase snake_case / Minúsculas con guion bajo |
| **Indentation / Indentación** | 2 or 4 spaces / 2 o 4 espacios |
| **Integrity / Integridad** | Always define PK & FK / Definir siempre PK y FK |
